The Mission

HexCombat is not just a game; it is an analytical instrument. Originally ported from TaiwanInvasionViewer, it serves two core purposes:

  • Primary: AI-vs-AI Research Instrument
    Enables headless batch games, generating Monte Carlo outcome distributions. It features an LLM JSON API allowing AI players to direct the simulation autonomously.
  • Secondary: Live-Adjudication Aid
    A facilitator can enter orders for both sides in the UI, letting the simulation resolve complex combat mathematics in real-time. Designed with a projector-friendly display.

Deep Simulation Mechanics

A rigorous port of the TIV mathematics. The Amphibious Offload system models the primary bottleneck of the invasion with exacting detail.

Capacity & Throughput

Day 1 assault maneuver BNs bypass throughput gates. Day 2+ forces compete for daily beach tonnage (2200t / BN). Capacity scales with floating piers and Jack-up barges, but is hard-capped by port availability (Auto-JLSF).

Not One Game — Thousands

One game is an anecdote. Because every game is deterministic under a seed — hierarchical, per-phase RNG, so the same seed reproduces the game exactly — HexCombat runs the same matchup across hundreds of seeds headlessly and reads the distribution of outcomes.

Parameter sweeps turn a single knob — beach capacity, mine density, CRBM maneuver — and measure how the outcome curve shifts. This is the primary mission: an instrument, not a match.

Honest Limits, Clear Roadmap

What it isn't (yet)

  • LLM play quality is bounded by the model — commanders reason locally, not campaign-deep.
  • Fidelity is the TIV port; new mechanics are deliberate, USER-directed changes, not free additions.
  • Live human-facilitator play is still landing (order entry, front-line draw).
