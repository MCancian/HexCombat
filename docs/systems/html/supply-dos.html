Supply — Red DOS Consumption (D2)

Human-readable summary. Machine reference: docs/systems/supply-dos.md. Port-fidelity audit, Area 4 of 10.

What this system does

Tracks the PLA's "Days Of Supply" (DOS) pool once brigades land. Each turn, every landed Red battalion burns supply tonnage that depends on how busy it was: a mechanized unit that both moved and fought burns the most; an idle, non-fighting foot unit burns the least. The tonnage is subtracted from the supply pool.

🟡 One port gap — supply doesn't affect combat yet.

TIV feeds each unit's real supply effectiveness into the combat strength calculation, so an out-of-supply unit fights weaker. HexCombat computes and drains the supply pool correctly, but currently passes a fixed 1.0 into combat — so running out of supply has no battlefield effect yet. The combat code already reads the field; only the population is missing. Logged as a port-completion item (deferred per the roadmap).
