Amphibious Offload (D1)

Human-readable summary. Machine reference: docs/systems/amphibious-offload.md. Port-fidelity audit, Area 3 of 10.

What this system does

Lands the PLA invasion force from ships onto Taiwan's beaches. On the first assault day, combat ("maneuver") battalions storm ashore in brigade-sized groups, limited only by how many brigade slots each beach has; their support battalions stay on the ships and wait. From day two on, everything left competes for the beach's daily tonnage throughput. A beach's capacity grows with floating piers and jack-up barges.

✅ Faithful port — verified.
  • Tonnage per battalion: 2200 — matches TIV exactly.
  • Beach capacity formula (base rate + floating piers + jack-up barges, ÷ 2200) — matches TIV's BeachThroughputService.
  • The 5 "maneuver" battalion types that land first — identical to TIV's list (Combined Arms, Amphibious Infantry, Mechanized Infantry, Air Assault Infantry, Special Forces).
  • Day-1 redesign behavior is covered by 54 automated tests mirroring TIV's own test cases.

Minor known divergences (intentional)

The ship-loading model is simplified vs TIV and these are documented in code / the port-audit list, not bugs:

  • Transport weight: every BN counts as 1.0 ship-equivalent; TIV uses a per-type transport-weight lookup.
  • Ship eligibility: any carrier can carry any BN; TIV splits amphibious vs cargo ship eligibility.

Both matter only if the anti-ship crossing (D3) is calibrated against exact ship manifests.
