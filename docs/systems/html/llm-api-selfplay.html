LLM API & Self-Play

Human-readable summary. Machine reference: docs/systems/llm-api-selfplay.md. Port-fidelity audit, Area 9 of 10.

What this is

A structured JSON interface so an AI agent can play the game with no UI: ask for an observation (the full board state from one side's view, plus the legal moves), then submit actions (move, commit, end turn). A built-in self-play harness runs both sides automatically for deterministic regression testing.

The contract

What
Observeobservation(team) → turn, phase, brigades, legal moves/commits, pending orders, last combat, supply, IJFS/anti-ship summaries, and game_over/winner.
Actapply_agent_response routes move / commit / end_turn. end_turn must include a seed — missing seeds are rejected (no hidden randomness).
Self-playSelfPlayRunner.play_game(policy, turns, seed) loops observe → decide → end-turn; the default policy is a zero-randomness reference. The policy is pluggable (swap in an LLM).

After the game: the briefing report

tools/make_game_bundle.py merges a finished game's record and replay log into one .viewer.json bundle (optionally adding 3-line commander SITREPs written by a local LLM) and can bake it into a single shareable game.html. tools/viewer/game_viewer.html presents it as a briefing: start at turn 1 and step forward with the mouse wheel, the ◀ ▶ buttons, or arrow keys — each step re-renders the hex map, reveals more of the trend charts (the full game's shape stays visible as a faint ghost), and swaps in that turn's SITREPs and adjudication. Charts: battalion census, cumulative PLA ship losses, and battalion losses per side per turn (China's split into ground combat vs drowned crossing).

Why it's solid

Two headless gates back it: one checks the observation keys, action application, and that a missing seed is rejected; another runs self-play twice with the same seed and asserts the two runs are byte-identical. Both are part of the standard test suite.

HexCombat-original. The original tool is a Flask web app with a browser UI — it has no agent/JSON API. This was designed from scratch for headless AI play. Deeper design lives in docs/LLM_PLAYTESTING.md, docs/LLM_OBSERVATION_SCHEMA.md, and docs/LLM_AGENT_PROTOCOL_PLAN.md; this page is the system map.
